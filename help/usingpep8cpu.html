Using Pep/8 CPU

Pep/8 CPU is a central processing unit simulator for writing microcode programs. It is designed to be used with the textbook, Computer Systems, J. Stanley Warford, Fourth edition, Jones and Bartlett, Publishers, 2010. The goal of the book is to teach the fundamentals of the classic von Neumann machine. You should use the Pep/8 system in conjunction with the textbook to write microcode programs.

Topics: Interactive and microcode use, Microcode files, Byte converter.

Interactive and microcode use

There are two ways to use the Pep/8 CPU simulator — interactive and microcode. When the application starts up, the two buttons Clock and Copy to Microcode are enabled and the two buttons Single Step and Resume are disabled.

In this state, you can experiment with the CPU one cycle at a time interactively. You can set individual control signals via the GUI widgets on the right side of the CPU pane. The A bus and B bus are colored red to indicate that the A and B adresses are set. When the user clicks the Clock button, the CPU pane shows the effect of this single CPU cycle.

Microcode files

The Pep/8 CPU system uses the file extension .pepcpu to store microcode programs. The files are text files that you can modify with your favorite text editor or word processor outside the Pep/8 CPU application.

When you write a microprogram the editor automatically numbers the statements that correspond to CPU cycles. The following figure shows the auto-cycle number feature of the editor. Comments and unit test specifications are excluded from the numbering.

When you save a microprogram the editor includes the cycle numbers in the text. For example, here are the first few lines of text from the file that is saved from the above figure. 

// File: fig1205.pepcpu
// Computer Systems, Fourth Edition
// Figure 12.5
// Fetch the instruction specifier and increment PC by 1

UnitPre: IR=0x000000, PC=0x00ff, Mem[0x00ff]=0xab
UnitPre: N=1, Z=0, V=1, C=0
UnitPost: IR=0xab0000, PC=0x0100
UnitPost: N=1, Z=0, V=1, C=0, T1=0x0a // T1 stores NZVC=1010

// Save the status bits in T1
1. CMux=0, C=11; LoadCk

// MAR <- PC, fetch instruction specifier.
2. A=6, B=7; MARCk
3. MemRead
4. MemRead, MDRMux=0; MDRCk
When you open a previously saved microprogram the editor strips any cycle numbers in the text and auto-renumbers them.

Byte converter

Any time the Pep/8 CPU application is running, the byte converter is available in the tool bar. The four input fields represent the value of a single byte as displayed from left to right in decimal, hexadecimal, binary, and ASCII. Simply enter any value in any input field and the other fields will show the same value in the corresponding format. Here is the result of entering the letter P in the ASCII field.
